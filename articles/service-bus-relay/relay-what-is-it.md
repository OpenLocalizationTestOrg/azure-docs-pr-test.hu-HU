# <a name="what-is-azure-relay"></a><span data-ttu-id="4957d-103">Mi az az Azure Relay?</span><span class="sxs-lookup"><span data-stu-id="4957d-103">What is Azure Relay?</span></span>

<span data-ttu-id="4957d-104">Az Azure Relay szolgáltatás úgy segíti a hibrid alkalmazásokat, hogy biztonságosan lehetővé teszi a vállalati hálózaton belüli szolgáltatások közzétételét a nyilvános felhőben anélkül, hogy meg kellene nyitni egy tűzfalkapcsolatot, vagy zavaró módosításokat kellene végrehajtani a vállalati hálózati infrastruktúrában.</span><span class="sxs-lookup"><span data-stu-id="4957d-104">The Azure Relay service facilitates hybrid applications by enabling you to securely expose services that reside within a corporate enterprise network to the public cloud, without having to open a firewall connection, or require intrusive changes to a corporate network infrastructure.</span></span> <span data-ttu-id="4957d-105">A Relay számos különböző átviteli protokollt és webszolgáltatási szabványt támogat.</span><span class="sxs-lookup"><span data-stu-id="4957d-105">Relay supports a variety of different transport protocols and web services standards.</span></span>

<span data-ttu-id="4957d-106">A továbbítási szolgáltatás támogatja a hagyományos egyirányú, a kérelem/válasz típusú és a társközi forgalmat.</span><span class="sxs-lookup"><span data-stu-id="4957d-106">The relay service supports traditional one-way, request/response, and peer-to-peer traffic.</span></span> <span data-ttu-id="4957d-107">Támogatja az események terjesztését is az internetes tartományban, így a pontok közötti nagyobb hatékonyság érdekében lehetővé válnak a közzétételi/előfizetési forgatókönyvek és a kétirányú szoftvercsatornás kommunikáció.</span><span class="sxs-lookup"><span data-stu-id="4957d-107">It also supports event distribution at internet-scope to enable publish/subscribe scenarios and bi-directional socket communication for increased point-to-point efficiency.</span></span> 

<span data-ttu-id="4957d-108">A továbbítón keresztüli adatátviteli mintában a helyszíni szolgáltatás csatlakozik a továbbítási szolgáltatáshoz egy kimenő porton keresztül, majd létrehoz egy adott szinkronizálási címhez kötött kétirányú szoftvercsatornát a kommunikációhoz.</span><span class="sxs-lookup"><span data-stu-id="4957d-108">In the relayed data transfer pattern, an on-premises service connects to the relay service through an outbound port and creates a bi-directional socket for communication tied to a particular rendezvous address.</span></span> <span data-ttu-id="4957d-109">Az ügyfél ezután kommunikálhat a helyszíni szolgáltatással, ha a szinkronizálási címet megcélozva forgalmat küld a továbbítási szolgáltatásnak.</span><span class="sxs-lookup"><span data-stu-id="4957d-109">The client can then communicate with the on-premises service by sending traffic to the relay service targeting the rendezvous address.</span></span> <span data-ttu-id="4957d-110">A továbbítási szolgáltatás ezután továbbítja az adatokat a helyszíni szolgáltatásba az egyes ügyfelek számára kijelölt kétirányú szoftvercsatornán keresztül.</span><span class="sxs-lookup"><span data-stu-id="4957d-110">The relay service then "relays" data to the on-premises service through a bi-directional socket dedicated to each client.</span></span> <span data-ttu-id="4957d-111">Az ügyfélnek nincs szüksége közvetlen kapcsolatra a helyszíni szolgáltatással, és azt sem kell tudnia, hol található, a helyszíni szolgáltatásnak pedig nincs szüksége megnyitott bejövő portra a tűzfalon.</span><span class="sxs-lookup"><span data-stu-id="4957d-111">The client does not need a direct connection to the on-premises service, it is not required to know where the service resides, and the on-premises service does not need any inbound ports open on the firewall.</span></span>

<span data-ttu-id="4957d-112">A Relay által biztosított kulcsfontosságú képesség a TPC-szerű szabályozással, végpontészleléssel, kapcsolati állapottal és átfedő végpontvédelemmel rendelkező, hálózati határok között történő kétirányú, nem pufferelt kommunikációban rejlik.</span><span class="sxs-lookup"><span data-stu-id="4957d-112">The key capability elements provided by Relay are bi-directional, unbuffered communication across network boundaries with TCP-like throttling, endpoint discovery, connectivity status, and overlaid endpoint security.</span></span> <span data-ttu-id="4957d-113">A továbbítási képességek abban térnek el az olyan hálózati szintű integrációs technológiáktól mint például a VPN, hogy a továbbítás egyetlen alkalmazásvégpontra is fókuszálható a gépen, míg a VPN technológia ennél jóval agresszívabb, hiszen a hálózati környezet módosítására támaszkodik.</span><span class="sxs-lookup"><span data-stu-id="4957d-113">The relay capabilities differ from network-level integration technologies such as VPN, in that relay can be scoped to a single application endpoint on a single machine, while VPN technology is far more intrusive as it relies on altering the network environment.</span></span>
