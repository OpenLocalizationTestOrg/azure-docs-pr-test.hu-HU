## <a name="devices-and-users"></a><span data-ttu-id="4051a-108">Eszközök és felhasználók</span><span class="sxs-lookup"><span data-stu-id="4051a-108">Devices and users</span></span>

<span data-ttu-id="4051a-114">A munkamenet a felhasználó által használt alkalmazás egy használati alkalma, amely az alkalmazás használatának a kezdetétől a befejezéséig tart.</span><span class="sxs-lookup"><span data-stu-id="4051a-114">A session is one use of the application performed by a user, from the time the user starts using it, until the user stops.</span></span>

<span data-ttu-id="4051a-115">A tevékenység az alkalmazás egy alrészének egy felhasználó általi egyszeri használata (ez általában egy képernyő, de az alkalmazás bármely megfelelő része lehet).</span><span class="sxs-lookup"><span data-stu-id="4051a-115">An activity is one use of a given sub-part of the application performed by one user (it is usually a screen, but it can be anything suitable to the application).</span></span>

<span data-ttu-id="4051a-117">A tevékenységek névvel azonosíthatók (legfeljebb 64 karakter hosszúságban), és tartalmazhatnak további adatokat (legfeljebb 1024 bájt mennyiségben).</span><span class="sxs-lookup"><span data-stu-id="4051a-117">An activity is identified by a name (limited to 64 characters) and can optionally embed some extra data (in the limit of 1024 bytes).</span></span>

<span data-ttu-id="4051a-118">A munkamenetek számítása automatikusan történik a felhasználó által végrehajtott műveletek sorozata alapján.</span><span class="sxs-lookup"><span data-stu-id="4051a-118">Sessions are automatically computed from the sequence of activities performed by users.</span></span> <span data-ttu-id="4051a-119">A munkamenet akkor kezdődik, amikor a felhasználó elindítja az első tevékenységet, és az utolsó tevékenység befejezésével ér véget.</span><span class="sxs-lookup"><span data-stu-id="4051a-119">A session starts when the user starts his first activity and stops when he finishes his last activity.</span></span> <span data-ttu-id="4051a-120">Ez azt jelenti, hogy a munkamenetet nem kell külön elindítani vagy leállítani.</span><span class="sxs-lookup"><span data-stu-id="4051a-120">This means that a session does not need to be explicitly started or stopped.</span></span> <span data-ttu-id="4051a-121">Helyette a tevékenységeket kell külön elindítani és leállítani.</span><span class="sxs-lookup"><span data-stu-id="4051a-121">Instead, activities are explicitly started or stopped.</span></span> <span data-ttu-id="4051a-122">Ha nincs jelentett tevékenység, jelentett munkamenet sincs.</span><span class="sxs-lookup"><span data-stu-id="4051a-122">If no activity is reported, no session is reported.</span></span>

## <a name="application-information"></a><span data-ttu-id="4051a-137">Alkalmazásadatok</span><span class="sxs-lookup"><span data-stu-id="4051a-137">Application information</span></span>
<span data-ttu-id="4051a-138">Az alkalmazásadatok a felhasználók címkézéséhez használhatók, vagyis adatoknak egy alkalmazás felhasználóihoz való társítására (a webes cookie-khoz hasonlóan, az alkalmazásadatok tárolása azonban a kiszolgálói oldalon történik az Azure Mobile Engagement platformján).</span><span class="sxs-lookup"><span data-stu-id="4051a-138">Application information (or app info) is used to tag users, that is, to associate some data to the users of an application (this is similar to web cookies, except that app info is stored on the server side on the Azure Mobile Engagement platform).</span></span>

<span data-ttu-id="4051a-139">Az alkalmazásadatok regisztrálhatóak a Mobile Engagement SDK API-jának vagy a Mobile Engagement platform eszköz API-jának használatával.</span><span class="sxs-lookup"><span data-stu-id="4051a-139">App info can be registered by using the Mobile Engagement SDK API or by using the Mobile Engagement platform Device API.</span></span>

<span data-ttu-id="4051a-140">Az alkalmazásadatok az eszközhöz rendelt kulcs/érték párok.</span><span class="sxs-lookup"><span data-stu-id="4051a-140">App info is a key/value pair associated to a device.</span></span> <span data-ttu-id="4051a-141">A kulcs az alkalmazásadat neve (legfeljebb 64 ASCII-betűből (a–z, A–Z), számokból (0–9) és aláhúzásjelből (_) állhat).</span><span class="sxs-lookup"><span data-stu-id="4051a-141">The key is the name of the app info (limited to 64 ASCII letters [a-zA-Z], numbers [0-9] and underscores [_]).</span></span> <span data-ttu-id="4051a-142">Az érték (legfeljebb 1024 karakter hosszú lehet) bármilyen karakterlánc, egész szám, dátum (éééé-hh-nn) vagy logikai érték (true vagy false) lehet.</span><span class="sxs-lookup"><span data-stu-id="4051a-142">The value (limited to 1024 characters) can be any string, integer, date (yyyy-MM-dd) or Boolean (true or false).</span></span>

<span data-ttu-id="4051a-143">Bármennyi alkalmazásadat társítható egy eszközhöz a Mobile Engagement díjszabási feltételei által meghatározott keretek között.</span><span class="sxs-lookup"><span data-stu-id="4051a-143">Any number of app info can be associated to a device, within the limits defined by the Mobile Engagement pricing terms.</span></span> <span data-ttu-id="4051a-144">Egy adott kulcsra vonatkozóan a Mobile Engagement a kulcsnak csak a legutóbbi értékét követi nyomon (az előzményeket nem).</span><span class="sxs-lookup"><span data-stu-id="4051a-144">For one given key, Mobile Engagement only keeps track of the latest value set (no history).</span></span> <span data-ttu-id="4051a-145">Az alkalmazásadatok értékének beállítása vagy módosítása arra kényszeríti a Mobile Engagement alkalmazást, hogy újraértékelje a célközönséghez kapcsolódó feltételek az alkalmazásadatok alapján (ha vannak), tehát az alkalmazásadatok használhatók valós idejű leküldések eseményindítóiként.</span><span class="sxs-lookup"><span data-stu-id="4051a-145">Setting or changing the value of an app info forces Mobile Engagement to re-evaluate audience criteria set on this app info (if any) meaning that app info can be used to trigger realtime pushes.</span></span>

## <a name="extra-data"></a><span data-ttu-id="4051a-146">További adatok</span><span class="sxs-lookup"><span data-stu-id="4051a-146">Extra data</span></span>
<span data-ttu-id="4051a-147">A további adatok olyan tetszőleges adatok, amelyek eseményekhez, hibákhoz, tevékenységekhez és feladatokhoz csatolhatók.</span><span class="sxs-lookup"><span data-stu-id="4051a-147">Extra data (or extras) is some arbitrary data that can be attached to events, errors, activities and jobs.</span></span>

<span data-ttu-id="4051a-148">A további adatok szerkezete hasonlít a JSON-objektumokhoz: a kulcs/érték párok fájából állnak.</span><span class="sxs-lookup"><span data-stu-id="4051a-148">Extras are structured similarly to JSON objects: they are made of a tree of key/value pairs.</span></span> <span data-ttu-id="4051a-149">A kulcsok legfeljebb 64 ASCII-betűből [a–z, A–Z], számból [0–9] és aláhúzásjelből (_) állhatnak, a további adatok teljes mérete pedig legfeljebb 1024 karakter lehet (miután a Mobile Engagement SDK JSON-formátumba kódolja azokat).</span><span class="sxs-lookup"><span data-stu-id="4051a-149">Keys are limited to 64 ASCII letters [a-zA-Z], numbers [0-9] and underscores [_]) and the total size of extras is limited to 1024 characters (once encoded in JSON by the Mobile Engagement SDK).</span></span>

<span data-ttu-id="4051a-150">A kulcs/érték párok teljes fájának tárolása JSON-objektumként történik.</span><span class="sxs-lookup"><span data-stu-id="4051a-150">The whole tree of key/value pairs is stored as a JSON object.</span></span> <span data-ttu-id="4051a-151">A kulcs/érték pároknak azonban csupán az első szintjét bontja fel a rendszer, hogy közvetlenül elérhetők legyenek néhány olyan fejlett funkció számára, amilyen például a Segments (Szegmensek) (egyszerűen definiálhat például egy „Sci-Fi-rajongók” szegmenset, amely az összes olyan felhasználót tartalmazza, akik legalább 10 alkalommal küldték el a „content_viewed” eseményt az elmúlt hónapban a „scifi” értékű „content_type” további kulccsal kiegészítve).</span><span class="sxs-lookup"><span data-stu-id="4051a-151">Nevertheless, only the first level of keys/values is decomposed to be directly accessible to some advanced functions like Segments (for example, you can easily define a segment called “SciFi fans” that is made of all users having sent at least 10 times the event named “content_viewed” with the extra key “content_type” set to the value “scifi” in the last month).</span></span> <span data-ttu-id="4051a-152">Ezért erősen ajánlott csak olyan egyszerű listákból álló további adatokat elküldeni, amelyekben a kulcs/érték párok skaláris értékek (például karakterláncok, dátumok, egész számok és logikai értékek).</span><span class="sxs-lookup"><span data-stu-id="4051a-152">It is thus highly recommended to send only extras made of simple lists of key/value pairs using scalar values (for example, strings, dates, integers or Boolean).</span></span>
